# 一、项目选题背景（第 1 页）

## 1.1 问题缘起

网约车已经成为城市出行的重要方式。在美国波士顿，Uber 和 Lyft 两家平台占据了几乎全部市场份额。它们的定价并非固定不变，而是采用**动态溢价（Surge Pricing / Prime Time）**机制——当某个区域的需求突然增加、可用车辆供不应求时，价格会自动上涨。

2018 年 11 月至 12 月，波士顿进入冬季，经历了多场暴风雪。极端天气下，出行需求与车辆供给之间的平衡被打破，溢价飙升成为常态。这引发了一系列值得探究的问题：

- 一场暴风雪来了，网约车价格多久会涨？涨多少？
- Uber 和 Lyft 谁更贵？谁先涨？谁涨得更猛？
- 不同区域之间的出行流向有什么规律？豪华车型和经济车型各占多少？

## 1.2 选题背景

本项目选取波士顿作为研究对象，基于以下考量：

1. **数据可得性**：Kaggle 公开了波士顿 2018 年 11–12 月 Uber/Lyft 完整行程数据
2. **天气多样性**：该时段涵盖晴天、阴天、雨天、暴雪等多类天气，便于分析天气对溢价的影响
3. **双平台竞争**：Uber 和 Lyft 在波士顿市场份额接近，具备对比分析的自然条件

# 二、面向的用户（第 2 页）

## 2.1 目标用户群体

| 用户类型 | 使用场景 | 关注重点 |
|----------|----------|----------|
| **普通乘客** | 出行前了解什么时候叫车更便宜、雨雪天溢价规律 | 价格趋势、最佳叫车时段 |
| **网约车司机** | 了解哪些区域、天气、时段溢价最高 | 溢价热力分布、高峰时段 |
| **平台运营/策略人员** | 对比分析竞争对手定价策略 | Uber vs Lyft 溢价差异 |
| **交通/城市研究者** | 研究网约车定价与天气、空间的关系 | 数据相关性、时空模式 |

## 2.2 用户需求抽象

所有用户的核心需求可以归纳为三点：

1. **看懂天气和溢价的因果关系**——什么时候溢价高？为什么高？
2. **对比两个平台的定价差异**——谁更贵？差距有多大？
3. **理解城市出行的空间模式**——人从哪里到哪里？坐什么车？

# 三、项目创新性与意义（第 3 页）

## 3.1 创新点

1. **天气 × 溢价的逐小时对齐分析**
   - 大多数网约车分析停留在日均或区域聚合，本项目将天气数据和溢价数据对齐到小时级别，在统一时间轴上呈现温度、降水、能见度与溢价的联动变化
   - 创新性地使用**事件对齐曲线**——将暴雪、暴雨等天气突变时刻对齐到 t=0，还原溢价响应的完整时间线

2. **Uber vs Lyft 全方位多维对比**
   - 不只是"谁更贵"，而是从溢价均值、稳定性、天气响应、高峰响应、夜间溢价、订单覆盖六个维度拆解差异
   - 使用 KDE 密度估计、热力矩阵、差异条形图等多种互补视角

3. **OD 流 + 车型分布的三维联动**
   - 弦图（拓扑关系） + 飞线地图（地理分布） + 车型堆叠图（产品构成），三种视角共享同一交互状态
   - 悬停/点击任意元素，三个视图同步高亮联动

4. **纯前端、零后端、全链路的可视化系统工程**
   - 从原始 CSV 到聚合 JSON 到可视化渲染，全部自动化
   - 入场动画、页面过渡、跨视图联动等交互细节打磨完整

## 3.2 项目意义

- **社会意义**：帮助乘客和司机理解动态定价机制，降低信息不对称
- **学术意义**：提供一个天气—溢价—空间三位一体的可视分析范式
- **工程意义**：展示现代前端技术栈（React 19 + D3.js + Leaflet）在可视化分析领域的工程实践

# 四、数据获取与处理方法（第 4 页）

## 4.1 数据来源

| 数据集 | 来源 | 规模 | 关键字段 |
|--------|------|------|----------|
| Uber/Lyft 行程数据 | Kaggle: Ride-sharing in Boston | ~600K+ 条行程记录 | 时间、起止经纬度、车型、价格、溢价倍数 |
| 天气数据 | 同期 Dark Sky API 历史数据 | 1,464 小时 | 温度、降水强度、能见度、天气类型、风速 |

## 4.2 数据处理流程

```
原始 CSV (367MB) → Node.js 聚合脚本 → 5 个结构化 JSON 文件 (共 ~6MB)
```

### 步骤一：小时级聚合
- 将 ~60 万条行程记录按小时分组
- 每小时计算：Uber/Lyft 各自的平均溢价、平均价格、订单量
- 关联同时段天气数据（温度、降水、能见度、天气摘要）

### 步骤二：OD 流聚合
- 根据经纬度将行程映射到 12 个核心区域
- 构建有向 OD 矩阵（12×12 = 144 条流）
- 合成差异化车型分布（7 种 Uber + 6 种 Lyft，基于距离偏斜因子）

### 步骤三：天气事件检测
- 检测天气突变事件（降水从无到有、温度骤降等）
- 提取事件前后 3 小时窗口数据
- 构建事件对齐响应曲线数据集

### 步骤四：天气分桶统计
- 按天气类型（晴、阴、雨、大雪等）对溢价数据进行分组统计
- 计算箱线图所需的均值、中位数、四分位数、异常值

### 最终输出文件

| 文件 | 用途 |
|------|------|
| `hourly_series.json` (590KB) | 时间河流图、密度曲线、散点矩阵 |
| `od-flow.json` (475KB) | 弦图、飞线地图、车型分布 |
| `weather_bucket_stats.json` (3.5KB) | 箱线图 |
| `event_windows.json` (2.4MB) | 事件对齐曲线 |
| `demand_blocks.json` (2.3MB) | 需求时空方块（扩展用） |

# 五、可视分析任务（第 5 页）

本系统的可视分析任务围绕三个核心问题展开：

## 任务一：天气如何影响网约车溢价？

- **时间维度**：在统一时间轴上展示温度、降水、能见度和 Uber/Lyft 溢价的逐小时变化
- **事件维度**：将天气突变时刻对齐，分析溢价响应的时间滞后和幅度
- **相关性维度**：量化各气象指标与溢价的统计相关性

**对应视图**：时间河流图、散点矩阵、事件对齐响应曲线

## 任务二：Uber 和 Lyft 的定价策略有何差异？

- **分布维度**：比较两平台溢价的整体分布形状（均值、方差、偏度、尾部厚度）
- **环境维度**：在不同天气条件、时间段下对比两平台的表现
- **多维维度**：从溢价均值、稳定性、天气响应等六个维度系统对比

**对应视图**：密度曲线、热力矩阵、天气柱状图、散点图、六维差异图

## 任务三：波士顿区域的出行流向和车型构成是怎样的？

- **拓扑维度**：12 个区域间有向流的规模和方向（弦图）
- **地理维度**：流向在真实地图上的空间分布（飞线地图）
- **产品维度**：每条流上各车型的占比分布（车型堆叠图）

**对应视图**：弦图、飞线地图、车型分布堆叠图

# 六、系统整体介绍（第 6 页）

## 6.1 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 前端框架 | React 19 + TypeScript | 组件化架构、类型安全 |
| 可视化 | D3.js v7 | 所有图表的手动绘制（弦图、KDE、时间序列等） |
| 地图 | Leaflet + React-Leaflet v5 | 飞线地图真实地理底图 |
| 动画 | Framer Motion | 入场动画、页面过渡、按钮交互 |
| 构建 | Vite 8 | 极速开发构建 |
| 部署 | Cloudflare Pages | 全球 CDN 静态托管 |

## 6.2 系统架构

```
入口（Landing）→ 总览（Overview）→ 三大分析模块

Landing（入场动画页）
  ↓
总览 Dashboard（Overview）
  ├── 全时段溢价走势（MiniTrendChart）
  ├── 日历热力图（CalendarHeatmap）
  ├── 天气类型分布（WeatherDonut）
  └── 快速导航卡片（→ 三个分析模块）

分析模块 A：天气溢价（Weather Surge）
  ├── 时间河流图（TimeRiverChart）
  ├── 散点矩阵（ScatterMatrix）
  ├── 箱线图（WeatherBucketBoxplot）
  └── 事件对齐曲线（EventAlignedCurve）

分析模块 B：平台对比（Platform Compare）
  ├── 密度曲线（DensityCurves）
  ├── 天气柱状图（WeatherBarChart）
  ├── 热力矩阵（HeatmapMatrix）
  ├── 散点对比（ScatterCompare）
  └── 六维差异图（DiffBarChart）

分析模块 C：流向与车型（OD Flow）
  ├── 弦图（ChordDiagram）
  ├── 飞线地图（OdFlowMap）
  └── 车型分布（VehicleStackChart）
```

## 6.3 交互设计

- **跨视图联动**：所有视图订阅同一交互上下文（OdFlowContext / WeatherSurgeContext）。悬停或点击弦图的弦带 → 飞线地图对应流向高亮 → 车型分布更新为该流向的数据
- **时间刷选**：在全时段走势图或时间河流图上拖拽框选时间段 → 全局联动到天气溢价分析页
- **键盘交互**：按 Esc 清除全部选中
- **响应式布局**：12 列 Bento Grid + 弹性组件尺寸

# 七、各视图介绍（第 7–10 页）

## 第 7 页：总览面板（Overview）

### 全时段溢价走势（MiniTrendChart）
- D3.js 绘制的双线时序图
- 蓝线 = Uber 平均溢价，橙线 = Lyft 平均溢价
- 支持 D3 brush 框选时间段，框选后联动到"天气溢价"页
- 悬停十字线 + 详细 tooltip

### 日历热力图（CalendarHeatmap）
- 11 月和 12 月的日历视图，并排对称居中
- 每个格子上半 = Uber 日均溢价，下半 = Lyft 日均溢价
- 颜色深度表示溢价高低（YlOrRd 色阶）
- 可看到 12 月中旬深红区域 = 暴风雪时期

### 天气类型分布（WeatherDonut）
- 环形图展示 10 种天气类型的时长分布
- 多云和阴天占多数，雨雪天虽少但贡献了绝大多数高溢价时段

---

## 第 8 页：天气溢价分析（Weather Surge）

### 时间河流图（TimeRiverChart）——核心创新视图
- **上半部分**：气象带，三层堆叠面积图展示温度（红蓝渐变）、降水强度（蓝色）、能见度（灰色）
- **下半部分**：Uber 和 Lyft 的溢价曲线，含散点标记高溢价时刻
- **底部**：D3 brush 刷选控件，可框选任意时间段放大
- **设计亮点**：天气和溢价在统一时间轴上，一眼看到暴雪来临时溢价如何飙升

### 散点矩阵（WeatherSurgeScatterMatrix）
- 每个小图 = 一个气象指标 vs 溢价
- 自动计算 Pearson 相关系数，颜色深浅表示相关性强度

### 天气分桶箱线图（WeatherBucketBoxplot）
- 按天气类型分组，展示 Uber/Lyft 各自的溢价分布
- 箱体中位数、四分位数、离群点一目了然

### 事件对齐响应曲线（EventAlignedCurve）
- 所有天气突变事件对齐到 t=0，向前后各展 3 小时
- 实线 = 平均溢价，半透明带 = 波动范围
- 直观展示溢价响应的滞后时间（约 1–2 小时）和峰值幅度

---

## 第 9 页：平台对比（Platform Compare）

### 溢价分布密度曲线（DensityCurves）
- KDE 核密度估计，两平台分布曲线叠加对比
- 虚线标注各自均值，右侧"长尾"反映极端溢价频率
- 内置结论：Uber 标准差更大，极端情况推价更猛

### 不同天气下的溢价对比（WeatherBarChart）
- 分组柱状图，按溢价高低从左到右排列天气类型
- 雪天和暴雨远高于其他天气

### 时段 × 天气溢价差热力矩阵（HeatmapMatrix）
- 横轴 24 小时、纵轴天气类型，颜色 = Lyft − Uber 溢价差
- 红色 = Lyft 更贵，蓝色 = Uber 更贵
- 右下角（深夜+雨雪）最红——Lyft 在这场景加价最猛

### Uber vs Lyft 溢价散点（ScatterCompare）
- 每点 = 一小时的 Uber/Lyft 溢价
- 对角线 = 两平台同价，上方 = Lyft 更贵
- 不同天气用不同颜色标注

### 六维差异对比图（DiffBarChart）——替代传统雷达图
- 六个维度对比：平均溢价、溢价稳定性、恶劣天气响应、高峰响应、夜间溢价、订单覆盖
- 蓝色条向左 = Uber 更强，橙色条向右 = Lyft 更强
- 基于实际数据计算百分比差异，不做归一化，有真实区分度

---

## 第 10 页：OD 流向与车型分布（OD Flow）

### 弦图（ChordDiagram）
- D3 chordDirected 布局，12 个区域的外圈弧段
- 中间弦带 = 有向 OD 流，粗细表示订单量
- 三种着色模式切换：溢价倍率 / 均价 / 订单量
- 悬停/点击联动飞线地图和车型分布

### 飞线地图（OdFlowMap）
- Leaflet 真实底图 + D3 SVG 叠加层
- 曲线 = OD 流向，粗细 = 流量，颜色（绿→橙）= 溢价高低
- 圆圈 = 区域节点，大小 = 总进出订单量
- 支持拖拽缩放，悬停/点击高亮联动

### 车型分布堆叠图（VehicleStackChart）
- 水平条形图，展示选中 OD 流或全部流的车型构成
- Uber 系列（蓝色系）7 种车型
- Lyft 系列（粉色系）6 种车型
- 虚线分隔 Uber/Lyft，显示占比和绝对数量

# 十、主要贡献与未来展望（第 13 页）
